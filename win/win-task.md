Конкурсное задание компетенции "Сетевое и системное администрирование" чемпионата ReaSkills 2026  

Модуль В: «Пуско-наладка инфраструктуры на основе ОС семейства Windows»  

ВАЖНО!!! По требованию ИБ в ходе выполнения задания вы не должны во внешней сети использовать имена внутреннего домена reaskills2026.local. Это касается DNS-имен, либо имен, используемых в сертификатах. Также запрещено менять топологию сети и делать внутреннюю сеть доступной извне – штраф за это обнуление всех технологий, касающихся внешней сети.  

1. Загрузите и установите групповые политики домена по управлению Microsoft Edge.  
   • Через GPO настроить для клиентских АРМ домена (данных и будущих). Т.е при добавлении нового АРМ (Windows 10/11 и только таких) в домен политика должна применяться автоматически без дополнительных действий со стороны администратора:  
      o Стартовая страница должна быть установлена – https://rds.reaskills2026.local  
      o Отключить приветственное окно предварительной настройки браузера Microsoft Edge  
      o Отключить анимацию при первом входе пользователя в систему  
      o Победите Дизн – уберите его из по-умолчанию открывающейся страницы  
2. На сервере DC  
   • Разверните корпоративный центр сертификации Certificate Authority  
      o Имя CA сделайте REA2026-CA  
      o Обеспечьте публикацию CRL по адресу http://crl.rea2026.ru из внешней сети и возможность просмотра списка, а также их обновление и скачивание новых  
3. На сервере RDS  
   • Установите приложение – мессенджер MAX  
   • Сделайте веб-интерфейс RDS доступным только по протоколу HTTPS (при попытке явно использовать протокол HTTP должна быть перенаправлена или отказ в открытии какой-либо страницы) и по строке адреса https://rds.reaskills2026.local  
   • Обеспечьте SSO веб-интерфейса RDS для пользователей, входящих с клиентских АРМ внутренней сети (проверка из внутренней сети будет осуществляться с АРМ - CLI-INT). При входе не должно быть требования ввода логина и пароля, а также никаких предупреждений и ошибок.  
   • Поместите ярлык файл со входом на опубликованное на RDS приложение MAX по протоколу RDP на рабочий стол обычных пользователей. Вход из внутренней сети не должен сопровождаться вводом пароля или наличием предупреждений, либо ошибок.  
4. На сервере ADFS разверните Active Directory Federation Service  
   • Имя портала ADFS должно быть REA ADFS  
   • Включите обязательную проверку списков отзыва сертификатов  
   • Включите страницу регистрации на портале ADFS  
5. На сервере WAP установите и настройте Web Application Proxy  
   • Опубликуйте веб-интерфейс RDS для доступа из внешней сети по адресу https://rdweb.rea2026.ru. Аутентификация должна осуществляться через форму ADFS. При открытии интерфейса как ADFS, так и RDS не должно быть ошибок и предупреждений. Через веб-интерфейс RDS мы должны иметь возможность скачать rdp файл мессенджера MAX  
   • Положите файл rdp файл подключения к мессенджеру MAX на рабочий стол АРМ (проверка из внешней сети будет осуществляться с АРМ - CLI-EXT)  
   • Подключение через rdp-файл из внешней сети должно быть через ввод доменного пароля пользователя.  
   • Добавьте возможность авторизации из внешней сети на Web Application Proxy через сертификат, выданный корпоративным центром сертификации. Выполните любой сертификат пользователя домена и установите на АРМ CLI-EXT.  

Интернет доступен со всех устройств сети, кроме CLI-EXT (он там не нужен, но настроить может сам). Сетевая связность настроена, но подразумевается, что входе выполнения задания её нужно будет дополнительно настраивать согласно заданию. Нарушение сетевой связности – проблема участника. В домене создана административная учетная запись reaskills2026\Administrator с паролем P@ssw0rd, а также 300 учетных записей пользователей с паролем P@ssw0rd1. Для проверки выполнения заданий группа проверки выберет 1-2 из 300 учетных записей. Внешний домен компании – rea2026.ru. Имена его зон разрешаются на сервере DNS RRAS1. Также для ускорения выполнения задания часть ролей, нужных по заданию, на сервере уже установлены.
